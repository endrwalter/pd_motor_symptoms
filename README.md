##  Overview
Predicting new-onset motor fluctuations and levodopa-induced dyskinesias (LID) is crucial for optimizing Parkinson’s disease management. This repository contains the machine learning methodology and evaluation framework developed to conduct a rigorous "stress-test" on the prognostic value of baseline clinical features. 

Due to data privacy regulations, the raw clinical dataset cannot be shared. Instead, this repository focuses on total methodological transparency, providing the complete codebase used to:
* Execute the **Nested Grid Search Cross-Validation** pipeline (`code_ensemble`).
* Configure and run multiple ablation studies to assess the impact of training cohort composition.
* Compute threshold-dependent (MCC, F1, Sensitivity, PPV/NPV) and threshold-independent (AUC-ROC) metrics.
* Extract feature importance and calibration metrics to align predictions with clinical knowledge.

## Usage Instructions

Because the clinical data is restricted, the repository is designed to showcase the exact scripts and configurations used to generate our findings.

**1. Reviewing the ML Pipeline:**
The core methodology is driven by configuration files (`.ini`) and executed via `main_calibrate.py`. You can inspect these files in the `code_ensemble/` directory to see the exact hyperparameters, cross-validation setup, and features used for predicting LID and Motor Fluctuations.

**2. Exploring the Results:**
The `results/` folder contains the detailed outputs (CSVs and PNGs) generated by our models during the various experimental conditions (e.g., random patient drop vs. targeted drop). 

**3. Interactive Post-Hoc Analysis:**
To understand how we derived our final conclusions and evaluated threshold-dependent vs. threshold-independent metrics, open the Jupyter notebooks in `code_other_analysis/`:
```bash
cd code_other_analysis
jupyter notebook explore_results_dyskinesia_rd.ipynb
```

## 📊 Data Availability
